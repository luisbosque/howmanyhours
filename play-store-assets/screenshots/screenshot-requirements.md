# Screenshot Requirements for Google Play Store

## Technical Requirements
- **Minimum**: 2 screenshots
- **Maximum**: 8 screenshots  
- **Format**: JPG or PNG
- **Resolution**: 16:9 or 9:16 aspect ratio
- **Size**: 1080 x 1920 pixels (portrait) or 1920 x 1080 pixels (landscape)
- **File size**: Max 8MB each

## Recommended Screenshots to Take

### Screenshot 1: Main Screen - Project List
**Show:**
- List of projects with monthly time summaries
- Clean interface
- App title/branding visible
- At least 2-3 example projects

### Screenshot 2: Active Tracking
**Show:**  
- Active project card with timer running
- Current run time display
- Monthly hours summary
- Stop and discard buttons
- Background notification visible

### Screenshot 3: Project Management  
**Show:**
- Creating a new project dialog
- OR project selection interface
- Clean, simple UI elements

### Screenshot 4: Time Entry
**Show:**
- Manual time entry dialog
- Minutes input field
- Simple, focused interface

### Screenshot 5: Monthly Summary (Optional)
**Show:**
- Project with significant monthly hours
- Time formatting examples
- Professional appearance

## Tips for Screenshots:
1. **Use realistic data**: "Client Work", "Study Time", "Exercise" as project names
2. **Show meaningful hours**: 15h 30m, 8h 45m rather than 0h 0m
3. **Clean status bar**: Remove notifications, show good battery/signal
4. **Consistent branding**: Keep app appearance uniform
5. **Good lighting**: Use device with good screen brightness
6. **Portrait orientation**: Most users browse Play Store in portrait

## Screenshot Captions (Optional, 30 characters max):
- "Simple project tracking"
- "One-tap time recording"  
- "Monthly time summaries"
- "Manual time entry"
- "Clean, focused design"

## Tools to Create Screenshots:
- Device screenshots (Power + Volume Down)
- Android Studio Device File Explorer
- Emulator screenshots (if needed)
- Screenshot editing tools for annotations/highlights
